## 📁 Repository Directory Structure

Every case file is contained in its own clean directory, featuring a dedicated project notebook alongside a specific `README` detailing the technical approach and mathematical formulas used:

* 📂 `01_corrupted_logistic_log` — Cleansing corrupted logistics array data and implementing exception handling for negative values.
* 📂 `02_duplicate_loyalty_fraud_detection` — Utilizing high-performance hashing (`set()`) for $O(1)$ constant-time duplicate credential lookups to secure retail loyalty systems.
* 📂 `03_ecommerce_metrics_engine` — Building reusable business intelligence functions to compute platform total revenue and Average Order Value (AOV).
* 📂 `04_server_log_error_miner` — Mining text server files using stable delimiter splitting (` - `) to isolate infrastructure error rates.
* 📂 `05_islamabad_tech_talent_pool_analytics` — (Cap Case) Engineering multi-field structural string parsing and dynamic frequency trackers.
* 📂 `06_islamabad_property_valuation_matrix` — Vectorizing mathematical formulas over plot sizes and sector multipliers using **NumPy** arrays to eliminate slow loops.
* 📂 `07_islamabad_tech_salary_remote_work_analyzer` — Implementing boolean masking, data slicing, and matrix aggregates over **Pandas** DataFrames.
